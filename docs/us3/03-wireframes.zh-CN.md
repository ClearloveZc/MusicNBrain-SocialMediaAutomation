# US3 低保真线框

负责人：Zhenchao  
版本：v1.0  
日期：2026-02-07

以下线框定义 US3 MVP 的教师端最小可用流程。

## 页面 A - 活动创建

```text
+------------------------------------------------------------------+
| 创建推广活动                                                      |
+------------------------------------------------------------------+
| 音乐会标题            [____________________________________]      |
| 时间                  [___________]  地点/链接 [___________]      |
| 学生亮点              [____________________________________]      |
| 报名链接              [____________________________________]      |
|                                                                  |
| [生成内容]                                                       |
+------------------------------------------------------------------+
```

主操作：`生成内容`  
校验：标题 + 时间 + 地点/链接为必填。

## 页面 B - 内容审核与编辑

```text
+------------------------------------------------------------------+
| 生成内容                                                          |
+------------------------------------------------------------------+
| 海报标题                                                         |
| [可编辑文本区域..............................................]    |
|                                                                  |
| Caption - TikTok                                                 |
| [可编辑文本区域..............................................]    |
| Caption - 小红书                                                 |
| [可编辑文本区域..............................................]    |
| Caption - Instagram                                              |
| [可编辑文本区域..............................................]    |
|                                                                  |
| [重新生成]   [保存草稿]   [下一步：排程]                         |
+------------------------------------------------------------------+
```

主操作：`下一步：排程`  
关键点：排程前必须保留人工编辑检查点。

## 页面 C - 发布排程

```text
+------------------------------------------------------------------+
| 排程发布                                                          |
+------------------------------------------------------------------+
| 系统建议计划（自动）：                                            |
|  - T-7 天   [datetime]  平台 [TT] [XHS] [IG]                    |
|  - T-3 天   [datetime]  平台 [TT] [XHS] [IG]                    |
|  - T-1 天   [datetime]  平台 [TT] [XHS] [IG]                    |
|                                                                  |
| 教师可修改每个时间点和平台选择。                                  |
|                                                                  |
| [返回]   [保存草稿]   [确认并入队]                               |
+------------------------------------------------------------------+
```

主操作：`确认并入队`

## 页面 D - 发布状态

```text
+------------------------------------------------------------------+
| 活动发布状态                                                      |
+------------------------------------------------------------------+
| Job ID     平台      时间                 状态        操作         |
| job-101    TikTok    2026-02-10 19:00     success     查看         |
| job-102    XHS       2026-02-10 19:00     failed      重试         |
| job-103    IG        2026-02-10 19:00     queued      --           |
|                                                                  |
| 错误详情（选中失败任务）：                                        |
| [token 已过期 - 请重新绑定账号后重试]                            |
|                                                                  |
| [重试失败任务]                                                    |
+------------------------------------------------------------------+
```

主操作：`重试失败任务`

## 交互说明

1. 活动在确认排程前始终保持 `draft` 状态。
2. 所有生成内容在入队前都允许二次编辑。
3. 重试默认沿用原 payload，除非用户明确修改内容后再发起。
